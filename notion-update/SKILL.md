# Notion 文档批量数据更新

输入：$ARGUMENTS（Notion 页面 URL 或 ID，以及要更新的内容描述；留空则逐步询问）

## 用途

批量修改 Notion 文档中的数据、参数、表格、计算结果。适用于基准参数变更后需要同步更新整篇文档的场景。

---

## 使用前必读：Notion MCP API 的关键限制

### ✅ 可以做的

- 用 `notion-update-page` + `update_content` + `content_updates` 替换文档中的文本字符串
- 用 `replace_all_matches: true` 批量替换同一字符串的所有出现位置
- 替换段落文字、表格内纯文本内容

### ❌ 无法做的

- **跨单元格匹配**：`old_str` 不能跨 `<td>` 边界，即不能用 `"25K | 1,750"` 这类字符串匹配表格中相邻两列的内容
- **HTML 标签匹配**：`old_str` 中不能含 `<td>`、`<tr>` 等 HTML 标签，必须是纯文本
- **单元格独立性**：表格每个单元格是独立 block，只能逐单元格用其唯一内容匹配

## 第一步：收集信息

如果 $ARGUMENTS 为空，依次询问：

> "需要几个信息来更新文档：
> 1. **Notion 页面 URL**：要修改哪个页面？
> 2. **变更内容**：哪些参数/数据需要更新？（提供旧值→新值列表）
> 3. **是否有派生计算**：修改基准参数后，是否需要重新计算依赖值？"

先用 `notion-fetch` 读取页面完整内容，再制定替换计划。

---

## 第二步：制定替换计划

读取页面后，列出所有需要替换的字符串对，格式如下：

```
替换清单：
1. "旧文本A" → "新文本A"（出现X次，replace_all: true/false）
2. "旧文本B" → "新文本B"（出现1次）
...
```

**制定顺序原则（防止链式污染）：**
- 替换时从**具体→模糊**：先替换长字符串/有上下文的字符串，再替换短字符串
- 替换时从**大值→小值**：如替换 `750K` 前先确认不会被 `50K→25K` 的替换污染（`750K` 含子串 `50K`）
- 如 A 是 B 的子串，必须先替换 B，再替换 A
- 有 Markdown 加粗的字符串（`**~2,100**`）和无加粗的（`~2,100`）是不同字符串，分开处理

---

## 第三步：执行替换

每次 `content_updates` 调用注意：

### JSON 转义规则

- `content_updates` 必须是 **array**，不是 string
- 中文字符直接写，不要用 `\uXXXX` 转义（`\u7` 是非法 Unicode，导致 JSON 解析失败，整个参数降级为 string）
- 如必须转义，Unicode 转义必须是完整 4 位十六进制：`\u57fa\u51c6`（基准），不能写 `\u57fa\u51c6`

### 替换策略

```
单次出现 → replace_all_matches: false（默认）
多次出现且新值相同 → replace_all_matches: true
多次出现但新值不同 → 分开替换，每次用唯一上下文定位
```

### 每次替换后验证

替换完成后，如果有不确定的结果，用 `notion-fetch` 重新读取验证，再继续下一步。

---

## 第四步：处理表格数据

### 表格单元格替换原则

- 每个单元格的内容必须是该单元格的唯一文本（避免与其他单元格冲突）
- 如果同一数值在多个单元格出现，使用 `replace_all_matches: true` 或逐个替换（先替换有上下文的行）
- **无法通过 API 修改的情况**：如果某单元格内容与其他单元格完全相同且无法区分，告知用户需要手动修改

### 计算口径声明

在替换表格数值前，明确告知用户使用的计算口径：

```
月份 X 的注册量计算：
- 口径：峰值播放量（发布后2周内），不用累计值
- 公式：[具体公式]
- 结果：[数值]

说明：第1批次发布后，视频只有峰值播放，不存在历史累计，
因此第1个月必须用峰值而不是累计口径。
```

---

## 第五步：常见陷阱清单

替换前逐项检查：

| 陷阱 | 检查方法 | 处理方式 |
|------|---------|---------|
| 子串污染 | 检查 old_str 是否是其他字符串的子串 | 先替换长串 |
| 链式替换 | 替换 A→B 后，B 是否又等于另一个待替换的 old_str | 调整顺序 |
| 双重出现 | 同一数值因多次替换操作意外出现在多处 | `notion-fetch` 验证后再替换 |
| HTML 标签 | old_str 含 `<td>` 等标签 | 改为纯文本 |
| Unicode 转义 | `\uXXX`（不足4位）或 `\u7`（非法）| 直接写中文或完整4位转义 |
| 加粗格式 | `**值**` 和 `值` 是两个不同字符串 | 分开处理 |

---

## 第六步：汇报结果

替换完成后输出汇总：

```
## 更新完成

### 已成功替换
- [清单，每行一个替换项]

### 需手动修改（API限制）
- [描述无法自动替换的内容，以及如何在 Notion 中手动修改]

### 建议验证
- 打开 Notion 页面，检查以下数值是否正确：[关键数值列表]
```

---

## 注意事项

- 每次替换前先 `notion-fetch` 确认当前内容，不要基于假设操作
- 对于复杂计算（如月度时间线），先在对话中列出完整计算过程供用户确认，再执行替换
- 如果用户提供了新的基准参数，询问清楚所有派生参数的计算逻辑后再操作，不要自行假设
- 替换后如不确定，宁可 `notion-fetch` 多验证一次，不要盲目继续
